# za38-cli v0.1 详细实施计划

> 状态：已确认，待执行
> 日期：2026-07-14
> 定位：企业内部 Coding Agent CLI。表现层为 Bun + OpenTUI/React，Agent 内核为 Python + deepagents/LangChain/LangGraph。

## 1. 目标与范围

### 1.1 v0.1 交付目标

交付一个可在 macOS、Linux、WSL 与 Windows 上通过单条安装命令使用的 Coding Agent CLI：

```text
macOS / Linux / WSL: curl -fsSL <install.sh> | bash
Windows PowerShell:  irm <install.ps1> | iex
Windows CMD:         curl -fsSL <install.cmd> -o install.cmd && install.cmd && del install.cmd
```

用户只执行 `za38`，不需要预装、理解或维护 Python、uv、Node/Bun。CLI 能在当前工作区以受控方式读写代码、执行受 allowlist 限制的命令、使用子 Agent/MCP/技能，并通过 OpenAI 兼容网关访问企业模型。

### 1.2 本期保留能力

| 编号 | 能力 | v0.1 决策 |
|---|---|---|
| A1 | 本地编码工具 | 保留：文件读写、编辑、删除、glob/grep、shell、todo |
| A2 | Agent 编排 | 保留：通用 task 子 Agent、原生自定义子 Agent |
| A3 | 交互控制 | 保留：HITL 审批、ask_user、自动审批、shell allowlist |
| A4 | 上下文 | 保留：压缩、Git/工作区上下文、恢复状态 |
| A5 | 会话 | 保留：SQLite checkpoint、线程恢复与浏览 |
| A7 | 记忆与技能 | 保留：AGENTS.md、技能发现、`/remember`、`/skill:<name>` |
| A8 | JS 解释器 | 保留：QuickJS `js_eval`，仅安全 PTC 默认集 |
| A9 | MCP | 保留：stdio/SSE/HTTP、配置、环境变量插值、信任、OAuth |

### 1.3 本期裁剪与延期

- **不实现：** web search、fetch URL、ACP。
- **延期：** 目标/评分（goal/rubric）、企业审计与 hooks、远程 sandbox、更新/doctor、主题/通知/剪贴板/编辑器/媒体等辅助体验。
- **模型层裁剪：** 不复制 dcode 的多 Provider 自动发现、模型切换 UI、推理强度和凭证面板；只实现 OpenAI 兼容模型。
- **兼容性裁剪：** 只读取 za38 原生的 agents/skills/memory 路径；不兼容 `.agents` 或 `.claude` 目录约定。

## 2. 架构决策

### 2.1 分层职责

```text
┌─────────────────────────────────────────────────────────┐
│ Bun + OpenTUI/React                                      │
│ TUI、输入、Slash Commands、流式事件渲染、审批界面         │
├──────────────── JSON-RPC 2.0 / stdio ───────────────────┤
│ 版本化 Schema、请求校验、事件顺序、心跳、错误与终态       │
├─────────────────────────────────────────────────────────┤
│ Python Agent Kernel                                      │
│ 配置、OpenAI 网关、deepagents、会话、MCP、技能、工具      │
└─────────────────────────────────────────────────────────┘
```

- Node/Bun 仅负责表现层与客户端本地命令；Python 是 Agent 业务逻辑的唯一事实来源。
- Python 子进程 stdout **只能**输出 JSON-RPC JSONL；日志和诊断写入受控日志通道，不能直接继承到 OpenTUI 终端。
- Python 内核被作为子进程/sidecar 启动，不能要求用户机器存在系统 Python。

### 2.2 进程与并发模型

现有 server 在 `query` 中等待整个 `astream()` 完成，导致运行期间无法读取 `cancel` 与 `respond`。必须改为：

1. `query` 创建唯一 `run_id`，立即返回 accepted 响应。
2. server 为该 run 建立后台 `asyncio.Task`，保存在 `active_runs[thread_id]`。
3. stdin 主循环持续调度 `cancel`、`respond`、`shutdown` 等控制请求。
4. 一个 thread 同时只允许一个 active run；同一 thread 的新 query 必须被拒绝或先在客户端排队，v0.1 采用拒绝并提示用户等待/取消。
5. 所有 stdout 写入使用单一 async lock，避免多个 run 的 JSONL 帧互相穿插。
6. 每个 run 无论成功、取消、模型异常、工具异常或进程关闭，都必须发出一个且仅一个终态事件。

### 2.3 JSON-RPC v1 协议

协议 Schema 放在 `packages/protocol/schema/v1.json`；TypeScript 与 Python 分别根据同一 Schema 校验消息，并使用共享 fixture 做契约测试。

#### Node → Python 请求

| Method | Params | 说明 |
|---|---|---|
| `initialize` | client/version/cwd/config path | 握手，返回能力、解析后的非秘密配置摘要 |
| `query` | `thread_id?`, `message`, `run_id` | 启动一个 Agent run，立即 accepted |
| `cancel` | `thread_id`, `run_id` | 取消当前 run |
| `respond` | `thread_id`, `run_id`, `interrupt_id`, `decisions` | 恢复 HITL 或 ask_user interrupt |
| `reload` | 无 | 空闲时重载配置、Agent、技能、子 Agent、MCP |
| `threads.list` | filter/limit | 线程浏览 |
| `agents.list` | 无 | 列出原生子 Agent |
| `skills.list` | 无 | 列出技能 |
| `mcp.list` | 无 | 返回服务器与工具状态 |
| `shutdown` | 无 | 取消 run、关闭 MCP 会话和 checkpoint |

#### Python → Node 事件

所有事件都携带 `thread_id` 与 `run_id`，工具事件还携带稳定的 `tool_id`：

```text
run/started
message/delta
tool/started
tool/updated
tool/completed
plan/updated
approval/requested
question/requested
heartbeat
run/completed
run/cancelled
run/failed
log
```

- `run/completed` 携带 input/output token usage、耗时和最终状态。
- `approval/requested` 必须包含工具名、参数摘要、风险说明与 `interrupt_id`。
- `question/requested` 必须包含问题、选项、是否多选和 `interrupt_id`。
- `heartbeat` 在长工具执行期间定期发送，客户端以此判断进程存活；不以固定 120 秒静默作为错误。
- 所有事件按 run 内 sequence number 排序；TUI 可检测丢帧并显示受控错误。

## 3. Python Agent 内核

### 3.1 依赖与版本策略

- 固定并锁定 `deepagents`、`langchain`、`langgraph-checkpoint-sqlite`、`langchain-openai`、`langchain-quickjs`、`langchain-mcp-adapters`、`httpx`、`pydantic` 与 MCP/OAuth 必需依赖。
- 使用锁文件和 release wheelhouse 构建运行时；不在终端用户机器上解析 Python 依赖。
- 迁移 dcode 代码时记录来源文件、上游 commit、许可证信息和本地修改原因；不依赖不稳定的行号。

### 3.2 OpenAI 兼容模型配置

配置来源与优先级（从低到高）：

```text
~/.harness/config.toml
<workspace>/.harness/config.toml
HARNESS_* 环境变量
--config PATH
```

首版配置格式：

```toml
[model]
provider = "openai-compatible"
name = "enterprise-coding-model"
base_url = "https://gateway.example.internal/v1"
api_key_env = "HARNESS_API_KEY"
timeout_seconds = 120
max_retries = 2

[model.headers]
X-Client-Name = "za38-cli"

[model.headers_env]
X-Enterprise-Tenant = "HARNESS_TENANT"
```

- 用 `ChatOpenAI` 构造模型；`api_key_env` 指向环境变量，TOML 和日志不得存储、回显或序列化明文密钥。
- `config show` 输出最终合并配置、来源和脱敏状态；`config path` 输出实际参与解析的路径。
- 缺少模型名、网关地址或密钥时，在 `initialize` 前返回可行动的错误，不启动 Agent。
- 不提供 `/auth`、`/model` 或配置写回；用户手工维护 TOML/环境变量。

### 3.3 Agent、工具和安全策略

- 基于 `create_deep_agent()` 构建，启用 filesystem、todo、subagents、summarization 与本地 shell backend。
- 增加/迁移：ConfigurableModel 的最小实现（仅配置加载）、ResumeState、AskUser、MemoryGuard、LocalContext、ShellAllowList、HITL、QuickJS、技能和 MCP 中间件。
- 工作区根目录由启动时 cwd 或 `--cwd` 决定。文件工具每次访问前解析真实路径，拒绝 root 外路径和越界 symlink。
- 交互模式下：写入、编辑、删除、execute、task、compact 都进入 HITL；只读工具自动允许。
- 无头模式下：默认只读；`--auto-approve` 才允许写入/编辑/删除/task；shell 仍必须显式提供 `--shell-allow-list`。
- `js_eval` 默认仅暴露安全只读 PTC；不得通过 JS bridge 绕过 HITL 或 shell allowlist。
- shell 仅能在工作区 cwd 中执行 allowlist 命令，并做路径/危险语法拦截；此为**尽力限制**，不是 OS 级隔离。系统提示词与文档必须明确这一限制。

### 3.4 子 Agent、记忆与技能

- 自定义子 Agent：`~/.harness/agents/<name>/AGENTS.md` 与 `<workspace>/.harness/agents/<name>/AGENTS.md`，使用 YAML frontmatter 描述名称、描述、可选模型和系统提示词。
- Agent 始终提供 dcode 的 general-purpose subagent；项目定义与用户定义同名时项目优先。
- 记忆路径：`~/.harness/AGENTS.md` 与 `<workspace>/.harness/AGENTS.md`；MemoryGuard 保护工具写入时的受管区域。
- 技能路径：内置 skills、`~/.harness/skills/<name>/SKILL.md`、`<workspace>/.harness/skills/<name>/SKILL.md`；不扫描 `.agents`、`.claude`。
- v0.1 仅提供 list/调用能力，不提供 CLI 创建、删除或信任管理命令；遇到无效 frontmatter、越界 symlink 或重复名称时跳过并通过 `log` 事件提示。

### 3.5 MCP

- 加载顺序：用户配置、项目配置、命令行 `--mcp-config`；`--no-mcp` 禁用全部加载。
- 支持 stdio、SSE 和 Streamable HTTP，配置环境变量插值，工具过滤以及工具名去重。
- 项目级 MCP 默认不可信：首次使用须在 TUI 通过审批事件确认服务器身份指纹；批准结果持久化并随配置变化失效。
- 保留 OAuth 登录与刷新能力。首版没有额外管理命令：`/mcp` 与 `za38 mcp list` 显示状态；首次遇到需登录服务器时由 TUI 发起交互式 OAuth 流程。
- 关闭/重载时关闭 MCP session manager，确保 stdio 子进程和 HTTP 会话释放。

### 3.6 会话与本地上下文

- SQLite checkpoint 保存在用户数据目录；使用 async saver 并在 server 生命周期内正确打开、drain、关闭。
- 每个 thread 保存工作区、分支、模型摘要、创建/更新时间和首条 prompt 摘要；`/threads` 和 `za38 threads list` 只读浏览、恢复线程。
- LocalContext 注入受控的 cwd、Git 状态、项目结构与 MCP 工具摘要；不得将环境变量、凭证或工作区外内容注入模型上下文。

## 4. Bun/OpenTUI 表现层与命令面

### 4.1 运行时与 TUI

- 官方运行时为 Bun，使用 vendored `@opentui/react` 和固定版本 `@opentui/core`；不承诺 Node 直接运行。
- TUI 包含：消息流、Markdown/代码块、工具卡片、计划视图、审批菜单、ask_user 表单、线程选择、Agent 选择、MCP 状态、输入框与状态栏。
- 事件 reducer 以 `thread_id/run_id/sequence` 维护状态，拒绝旧 run 或重复帧；工具卡片必须在错误/取消时结束为终态。
- Ctrl+C 优先调用 `cancel`；第二次 Ctrl+C 或 `/force-clear` 才强制结束本地子进程。关闭时先发 `shutdown`，随后等待有限时间并回收进程。

### 4.2 首版 Slash Commands

仅实现下列命令及必要别名：

```text
/help
/quit (/q)
/clear
/force-clear
/threads
/compact
/mcp
/tools
/reload
/remember
/skill:<name>
/agents
/version
```

- `/clear` 新建 thread；`/force-clear` 先取消当前 run，再新建 thread。
- `/threads` 浏览与恢复，不提供删除。
- `/compact` 触发当前 thread 的摘要/offload。
- `/mcp` 展示服务器、信任、连接与 OAuth 状态；`/tools` 展示当前 Agent 的内置与 MCP 工具。
- `/reload` 仅在无 active run 时执行；若重建 Agent/MCP 失败，继续保留旧实例并展示错误。
- 不实现 `/auth`、`/model`、`/tokens`、`/restart`、`/goal`、`/rubric`、`/theme`、`/update`、`/install` 等 dcode 命令。

### 4.3 CLI 命令与参数

首版只读管理命令：

```text
za38 config show|path
za38 threads list
za38 agents list
za38 skills list
za38 mcp list
```

首版运行参数：

```text
--config PATH        --cwd PATH          --resume [THREAD_ID]
--agent NAME         --message TEXT      --non-interactive TEXT
--stdin              --json              --max-turns N
--timeout SECONDS    --auto-approve      --shell-allow-list LIST
--mcp-config PATH    --no-mcp            --interpreter / --no-interpreter
--version            --help
```

- 无头 JSON 输出仅写 stdout；诊断和进度写 stderr 或 JSON 事件，绝不混入最终结果。
- 不实现默认模型写入、模型参数覆盖、Provider extras、sandbox 参数、ACP、安装/更新/doctor 参数。

## 5. 跨平台安装与打包

### 5.1 发行物结构

每个版本发布经签名的完整平台 runtime 包，覆盖：

```text
darwin-arm64, darwin-x64,
linux-x64-gnu, linux-arm64-gnu,
linux-x64-musl, linux-arm64-musl,
win32-x64, win32-arm64
```

平台包包含：

```text
bin/za38                 # 用户唯一入口
runtime/bun              # 私有、固定版本 Bun
app/                     # 编译后的 CLI 与 OpenTUI 原生依赖
agent/za38-agent         # 打包后的 Python Agent sidecar 及资源
manifest.json            # 版本、平台、文件校验和、协议版本
```

- Python sidecar 在 CI 的目标平台上构建为 onedir 可执行发行物，包含 prompts、内置技能与全部锁定依赖；不得依赖系统 Python。
- `@za38/cli` 作为高级 npm 入口，透明解析同版本平台 runtime 包；用户仍只安装 `@za38/cli`。npm postinstall 只验证 runtime 选择和 manifest，不能下载 uv、Python 或 pip 依赖。

### 5.2 安装器行为

- Unix installer 识别 darwin/linux/WSL、CPU 架构与 libc，下载并校验匹配发行包，原子安装至用户数据目录，创建 `za38` 启动器并补充 PATH 提示。
- PowerShell/CMD installer 下载 Windows zip，安装到 `%LOCALAPPDATA%\\za38`，创建 `za38.cmd` 并更新用户 PATH。
- 安装器内置公开验签材料，验证 release manifest 签名和 SHA-256；校验失败不安装。
- 更新系统延期：重新执行安装器安装新版本，旧版本仅在新版本自检成功后清理。
- 所有下载域名、内部 npm registry、签名密钥和发布目录由 CI release 配置注入；仓库不硬编码真实企业 endpoint 或秘密。

## 6. 实施顺序与验收门槛

### Phase 0：修正现有基础

1. 固化 workspace 构建、锁文件、Python 锁定依赖与资源打包规则。
2. 修正当前 Python 测试替身：不能使用未实现 `bind_tools()` 的 `GenericFakeChatModel` 验证工具 Agent。
3. 把 server 改为并发控制面，完成 JSON-RPC v1 Schema 和双端契约 fixture。
4. 验收：initialize/query/stream/cancel/respond/shutdown 可由真实子进程端到端验证。

### Phase 1：最小可用 Agent 垂直切片

1. 实现 OpenAI 兼容模型配置、配置合并和脱敏 CLI 查询。
2. 组装 deepagents 本地 backend、文件工具、todo、HITL、ask_user、shell allowlist、QuickJS 安全 PTC。
3. 完成 OpenTUI 消息流、工具卡片、审批/提问界面和无头模式。
4. 验收：通过 mock OpenAI server 在工作区完成“读文件→申请写入→用户批准→写入→运行 allowlist 测试→完成”。

### Phase 2：持久能力

1. 集成 SQLite checkpoint、线程恢复、`/threads` 与 `za38 threads list`。
2. 集成 LocalContext、compact、MemoryGuard、原生 skills 和自定义子 Agent。
3. 验收：退出后恢复 thread；同名项目 Agent/技能按优先级覆盖；工作区外文件和 symlink 访问被拒绝。

### Phase 3：MCP

1. 加载配置、stdio/SSE/HTTP transport、工具过滤、session 生命周期。
2. 集成项目 MCP 信任指纹和 OAuth 交互事件。
3. 验收：可使用可信 stdio 与 HTTP MCP；未信任项目 MCP 不会启动；配置变化后要求重新批准；重载与退出均释放会话。

### Phase 4：完整交付与跨平台验证

1. 构建 Python sidecar、Bun/OpenTUI runtime 和平台包。
2. 实现三个安装器、签名 manifest、原子安装与卸载/重装测试。
3. 在 macOS、Linux glibc/musl、Windows runner 做安装与启动冒烟测试。
4. 验收：干净机器无需系统 Python/uv/Node/Bun 即可安装并运行 `za38 --version`、无头 mock 任务和交互 mock 任务。

## 7. 测试清单

- **协议：** 非法 JSON、未知 method、半帧、重复/乱序事件、并发 run、取消、HITL resume、stderr 隔离、子进程异常和终态唯一性。
- **模型与配置：** 四级合并优先级、缺失项错误、环境变量 header、密钥脱敏、OpenAI HTTP mock、超时/重试。
- **Agent：** 文件工具、审批路径、无头只读默认、`--auto-approve`、shell allowlist、QuickJS PTC、compact、子 Agent。
- **状态：** checkpoint 生命周期、恢复、线程筛选、取消后状态一致性。
- **Skills/MCP：** 原生目录发现、无效 frontmatter、symlink 拒绝、stdio/SSE/HTTP、OAuth mock、信任指纹失效、reload/cleanup。
- **TUI：** reducer、输入、工具状态、审批、ask_user、Slash Commands、窄终端渲染和无头 JSON。
- **分发：** manifest/签名失败、平台选择、PATH、重复安装、升级原子性、八类目标产物冒烟。

## 8. 后续接口预留

- 企业沙箱：实现 `ExecutionBackend` adapter，接管本地 backend 的文件、shell、同步和 diff；保持 Agent 工具名与 IPC 事件不变。
- za38 垂域能力：独立项目提供工具/技能包后，以 Python `tools` 注册和内置 skill 两种方式接入；不改变 TUI、协议或安装器。
- 企业鉴权、日志和审计：通过模型/执行 backend/MCP 的 middleware 插槽注入；v0.1 不发送外部追踪数据。
