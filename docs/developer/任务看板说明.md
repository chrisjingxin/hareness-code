# 任务看板说明

任务唯一来源是 `docs/developer/tasks/`：一项任务一个 Markdown 文件。`docs/developer/任务看板.md` 是只读生成物，禁止手工编辑。

## 状态与认领

状态仅可为：`待认领`、`进行中`、`阻塞`、`待验收`、`已完成`。认领时必须写入负责人和分支：

```bash
bun run task:claim -- ZC-001 --owner codex --branch codex/zC-001-tui
```

`进行中` 必须有负责人和分支；`已完成` 必须同时有测试证据、完成日期和用户/开发者文档影响记录。完成示例：

```bash
bun run task:complete -- ZC-001 --evidence "bun run test" --references "abc123"
```

## 元数据

每个任务的 front matter 固定包含：`id`、`title`、`priority`、`status`、`owner`、`branch`、`scope`、`acceptance`、`user_docs`、`developer_docs`、`test_evidence`、`references`、`completed_at`。优先级为 `P0`、`P1` 或 `P2`。

同步和校验：

```bash
bun run tasks:sync
bun run tasks:check
```

任务正文应描述范围、非范围和可验证验收条件。若文档引用任务 ID，`bun run docs:check` 会确认该 ID 存在。
