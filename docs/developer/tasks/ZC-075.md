## 范围

- 在受 WorkspaceBoundary 保护的写工具成功后记录 Agent turn、受影响文件、前置指纹、可逆 patch/快照、状态和过期策略；记录不包含完整秘密内容。
- 提供 `/diff [query]` Viewer 以及 `/undo`、`/redo` Action/确认流程，发生用户后续修改、路径逃逸、补丁冲突或缺失 Checkpoint 时 fail closed。
- 显示 Git 工作区摘要只能作为辅助，不把所有脏改动归因给 Agent；操作绝不调用 `git reset --hard`。
- 将 Checkpoint 生命周期与 EventLog/replay、thread 取消和存储恢复对齐。

## 非范围

- 不实现任意 Git 历史重写、跨仓库撤销、二进制大文件版本库或替代用户自己的版本控制工作流。
- 不让模型直接调用 undo/redo 或修改 Checkpoint 内容。

## 验收清单

- 连续修改、部分失败写入、用户并发编辑、符号链接、删除/新建、重启和 redo 失效均有安全回归。
- Diff 清楚区分归属并在不支持的内容上降级，不泄露工作区外路径。
- Undo/redo 在确认、取消和冲突下不破坏用户改动，审计事件与 UI 状态一致。
