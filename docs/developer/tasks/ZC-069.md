## 范围

- 将 `/skills [query]` 迁移为 ResourceManager：搜索、来源/版本/参数提示/启用状态详情、选择下一次任务及 capability-gated 的 inspect、enable、disable、install、update、remove 操作。
- 复用现有 `skills.*` JSON-RPC 和可信来源校验；破坏性或市场操作必须确认，失败显示脱敏诊断。
- 明确提示 catalog 与 PromptEpoch 快照：管理操作不改写当前或已存在 thread，按现有规则仅在之后的新 thread/新进程中生效。
- 保留 `/skill:<canonical-id>` 兼容；短名称只在全局唯一时可作为便捷入口。

## 非范围

- 不实现文件监听、运行中自动 reload、MCP Prompt 或 Plugin Command Loader。
- 不放宽 Skill 的路径、symlink、front matter、市场认证或项目可信边界。

## 验收清单

- 所有未协商 `skills.manage` 的操作隐藏或禁用，选择器仍能在只读 capability 下工作。
- 安装、启停、更新、移除、冲突名称、catalog 错误和取消均有 TUI/RPC 回归，且没有泄露资源路径或正文。
- 管理前后的活动 thread 使用相同 PromptEpoch，后续 thread 使用文档声明的 snapshot 语义。
