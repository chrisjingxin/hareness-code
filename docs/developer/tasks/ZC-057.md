## 范围

- 在 server 的 run.start、context.compact、cancel、thread 恢复和 shutdown 路径统一 acquire/release Runtime lease，不再缓存 `thread_id -> Agent` 或 `thread_id -> middleware`。
- 在 TOML v1 增加受限的 RuntimePool 配置：最大 Profile 数、空闲 TTL、关闭超时和可选 pin；默认值必须保守且有上限校验。
- 实施 LRU 容量淘汰、TTL sweep、配置失效后的旧 Runtime DRAINING 与新 Profile 构建；全部候选 Runtime 活跃时返回稳定的资源繁忙错误，禁止绕过池无限创建。
- sidecar 退出时停止新 run、协调取消/完成、关闭 Pool 后再关闭 ThreadStore；对 `context.compact` 保持现有空闲 thread 约束。

## 非范围

- 不增加新的 JSON-RPC capability、守护进程、多项目服务端或运行中热更新配置。
- 不实现真实 RSS 硬限制、OS 级内存回收、远端 Sandbox 或 MCP 热重载。
- 不改变单个 thread 同时只能有一个活动 run 的协议规则。

## 验收清单

- 创建大量相同 Profile thread 后，图构建数为一；不同模型/工具/Policy 配置得到隔离 Runtime。
- TTL 与 LRU 只淘汰 `IDLE` 且无 lease/run/queue 的 Runtime；活跃 run 在淘汰竞争下仍完成或按既定取消语义结束。
- 被淘汰 Runtime 的 thread 再次运行时从持久状态恢复，不丢失消息、PromptEpoch 或上下文 artifact。
- server shutdown、构建失败、compact、同 thread 拒绝并发和跨 thread 并发均有 Python 集成回归。
