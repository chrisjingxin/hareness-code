## 范围

- 定义并迁移 TeamStore 数据模型：team metadata、agent instance、assignment、mailbox message 与状态转换；每条消息记录 `thread_id`、`run_id`、`team_id`、发送/接收 Agent、kind、payload 摘要、causation/correlation ID、序号和终态。
- 提供只允许已登记 Team 成员使用的 direct、broadcast、request、result、handoff、cancel 消息 API，以及 payload/数量/保留期限限制。
- 将 mailbox 与 assignment 状态持久化到 ThreadStore 相邻的受权限保护存储；Runtime 和内存队列只是投递缓存，不能成为唯一事实来源。
- 定义重启/Runtime 淘汰后的恢复、去重和过期策略，为后续 EventLog 投影保留稳定内部事件模型。

## 非范围

- 不启动并行 worker，不实现协调器调度、TUI 面板或用户可调用的任意 Agent 消息接口。
- 不允许跨 project、跨 thread 或未授权 Team 的通信。
- 不把原始大工具输出、Prompt 或凭据复制进 mailbox payload。

## 验收清单

- Runtime 关闭和 sidecar 重启后，未解决 assignment 与未消费消息可按顺序恢复，已确认消息不会重复执行。
- 任意伪造的 team/agent/thread 归属、过大 payload、非法 kind、循环自投递和越权广播均被拒绝。
- 并发投递保持单 team 的顺序/幂等约束，且不同 thread/team 无交叉可见性。
- 数据库迁移、恢复、清理和脱敏诊断均有单元及集成回归。
