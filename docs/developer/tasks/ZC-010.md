## 范围

把已解析的 `approval_mode` 从配置层传入 Agent 工厂，定义并实现 `plan`、`default`、`auto-edit`、`yolo` 的稳定语义。

## 非范围

不新增永久批准、每工具白名单、企业权限中心或远端 sandbox Provider。

## 验收清单

- `default` 对受控写入、shell、删除和子 Agent 发起 HITL 请求。
- `auto-edit` 只自动放行文件编辑，shell 和高风险操作仍需审批。
- `plan` 禁止实际变更工具执行，并向模型返回可操作的计划模式结果。
- `yolo` 不产生 Harness HITL，但工作区边界等硬策略仍然生效。
- 无头客户端不协商交互能力，审批和问答均 fail closed。
- Python 与 Bun 测试同时断言后端行为和 TUI 展示一致。
