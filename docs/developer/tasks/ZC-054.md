## 范围

- 定义不可变 `RuntimeProfile` 与 canonical JSON / SHA-256 `RuntimeProfileKey`，包含项目边界、Agent topology 及版本、模型角色绑定、工具与可执行 Skill catalog、MCP、Sandbox、Policy、middleware 和 Prompt 模板版本。
- 明确哪些字段属于 Profile、Thread State 或 Run Context；`thread_id`、消息、PromptEpoch 正文、上下文摘要、待审批交互、run ID 与 mailbox 消息不得参与 Profile Key。
- 在 ThreadStore 保存可审计但脱敏的 Profile 摘要和版本，建立既有 thread 的兼容读取与默认迁移路径；不得重写或丢失 checkpoint、消息和 PromptEpoch。
- 为后续 Runtime、模型角色和 Agent Team 提供稳定 DTO，不在业务模块中散落 hash 规则。

## 非范围

- 不构建 AgentRuntime、RuntimePool、LRU/TTL 或修改 JsonRpcServer 的运行路径。
- 不引入多模型角色、Agent Team、MCP 热加载或新的用户可见协议能力。
- 不将完整项目路径、API Key、原始系统提示词或工具参数写入 ThreadStore、日志或 Profile 摘要。

## 验收清单

- 配置相同的不同 thread 得到完全相同的 Key；任一图结构/能力字段改变均得到不同 Key。
- Thread/Run 动态字段改变不会改变 Key，测试覆盖 canonicalization 与哈希稳定性。
- 旧数据库升级、服务重启和恢复旧 thread 后，原有消息、PromptEpoch、上下文状态与 checkpoint 均可读取。
- Profile 摘要、异常和诊断输出不含秘密、原始工作区路径或完整 Prompt。
