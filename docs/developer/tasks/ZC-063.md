## 范围

- 定义带 canonical ID、别名、来源、展示类型、参数提示、requirements、安全策略和 availability 的 `CommandDefinition`；建立只读 Registry、解析与搜索排序接口。
- 将 `/help`、菜单与补全改为由 Registry 投影；读取已协商 capability、当前 thread、活动 run 和待处理交互后统一返回 available、disabled 或 hidden。
- 实现大小写不敏感的命令解析、参数原文保留、`//` 转义、未知命令的拼写建议；`/continue` 必须成为 `/resume` 的真实别名，未知 `/command` 不得发送给模型。
- 为后续 Builtin、Skill、MCP、Plugin Loader 保留注册与冲突检测接口，但不在本任务加载这些新来源。

## 非范围

- 不增加新的业务命令、JSON-RPC 方法、TUI 弹窗或配置编辑能力。
- 不改变当前 Skill canonical ID、线程恢复范围或 Agent Prompt 语义。

## 验收清单

- 同一 Registry 的数据能生成菜单、帮助、补全和解析结果，不能再出现注册表外的特殊命令分支。
- 缺少 capability 的命令隐藏；运行中或无 thread 时的命令显示稳定禁用原因；规则有纯函数测试。
- `/HELP`、`/q`、`/continue`、引号参数、未知拼写和 `//` 都有回归测试，普通提示词仍原样提交。
