## 范围

- 从 `SkillPicker`、`ThreadPicker` 的重复遮罩、尺寸、搜索框、加载/错误/空态、列表滚动和选中逻辑中提取泛型 `SearchPicker<T>`；由调用方提供行渲染、稳定键、标题、搜索占位和选择回调。
- 提供最小的 Confirm/Dialog Shell，统一打开、关闭、焦点恢复、键盘/鼠标交互和结果回调，供 `/new`、`/compact` 等已有或紧邻动作使用。
- 统一焦点捕获与恢复、键盘/鼠标操作、加载/空/错误状态、背景压暗、终端尺寸变化和窄终端单列降级。
- 将 CommandResult Adapter 与该组件层连接，避免同类 Picker 在根组件重复维护浮层布局和输入生命周期。
- 迁移现有 `/resume` 和 `/skills` 选择器，并为 ZC-067 的 `/model` Picker 固化可直接复用的接口。

## 非范围

- 不实现模型、MCP、权限、配置或新的 Thread 领域操作；`/model` 的数据和选择语义仍由 ZC-019、ZC-067 交付。
- 不预先实现 ResourceManager、InspectorView、TimelinePicker、复杂表单或资源详情面板；这些抽象只能在出现至少一个实际调用方时随对应任务引入。
- 不改变 Markdown、语法高亮或非命令会话渲染。

## 验收清单

- SearchPicker<T> 与 Dialog Shell 有独立渲染/键盘回归；Esc 不改变草稿，焦点能返回 composer，异步错误不会留下遮罩或失效输入。
- 标准与窄终端中，Thread/Skill 选择器的已有行为、搜索和状态隔离回归通过，二者不再复制浮层布局实现。
- ZC-067 可通过已定义的 SearchPicker<T> 接口实现 Model Picker，无需修改基础组件的布局、焦点或尺寸逻辑。
