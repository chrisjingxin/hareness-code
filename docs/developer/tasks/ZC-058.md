## 范围

- 定义 RuntimePool 诊断快照和结构化日志字段：Profile Key、状态、active lease/run、命中/未命中、构建/关闭耗时、淘汰原因与关闭失败；日志不得包含 Prompt、凭据或原始路径。
- 在既有 `/status` 或等价本地诊断中以能力允许的脱敏摘要暴露运行池状态，并更新故障排查说明。
- 建立使用 mock graph/资源的确定性压力回归，覆盖相同/不同 Profile、大量历史 thread、TTL/LRU、DRAINING、关闭失败和服务重启恢复。
- 为未来真实内存监测预留指标接口，但使用稳定的 Runtime 数与资源关闭断言替代脆弱的 CI RSS 绝对值断言。

## 非范围

- 不接入 OpenTelemetry、外部指标后端、云监控或 crash report 服务。
- 不依据未校准的 RSS 直接杀死 Runtime，也不修改淘汰策略本身。
- 不引入 Agent Team 级指标；其由后续 TeamScheduler 任务补充。

## 验收清单

- 同 Profile 1000 个模拟 thread 的构建数保持为一；多 Profile 总 Runtime 数不超过配置容量。
- 每个 eviction 都有原因和关闭结果，失败资源不会阻止其他资源释放；重启后诊断不引用旧 Runtime。
- 压力回归验证 thread 的持久消息、PromptEpoch 和 context artifact 不交叉。
- `/status` 或等价诊断不泄露 Key、工作区原路径、Prompt 或模型凭据。
