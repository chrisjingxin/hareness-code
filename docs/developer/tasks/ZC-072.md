## 范围

- `/settings` 按 Model、Context、Permissions、Sandbox、Skills、MCP、UI、Logging 和 Enterprise 分类展示最终值、来源、覆盖关系、可编辑性与重启/新 thread 提示。
- `/permissions` 仅展示并选择现有审批模式及已实现的相关策略；界面必须说明本机/远端执行边界，不能把未实现的网络、工作区外访问或 MCP 规则伪装成可控制项。
- 使用预览、确认和结果通知调用 ZC-071 服务；受管理锁、未信任来源和缺少 capability 的字段只读可见并显示原因。
- `/config`、`/prefs`、`/permission` 只作为兼容别名，菜单展示 canonical 名称。

## 非范围

- 不提供任意键值写入、秘密编辑器、无确认的 yolo 提权或运行中强制改变活动 run 的策略。
- 不重新实现 ZC-042 的主题/终端偏好领域逻辑。

## 验收清单

- 用户、项目、managed、环境变量和默认来源的显示与锁定逻辑有端到端测试。
- 未知/旧审批模式安全降级，界面与 `approval_policy` 的真实执行结果一致。
- 取消、冲突、验证失败、活动 run 和窄终端下均不会残留半完成配置或错误焦点。
