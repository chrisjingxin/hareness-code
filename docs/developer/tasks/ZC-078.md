## 范围

- 将 `/model [query]` 的确认从“仅暂存给下一个新 Thread”改为受策略保护的模型切换操作：空闲时更新当前 Thread 的后续运行模型，并更新用户级默认 Profile，使之后新建的 Thread 持续继承该选择。
- 在 ThreadStore 中保存按生效顺序排列的脱敏角色绑定历史或等价事件记录；每个 run 可追溯实际 executor Profile、Provider 标签、模型名、能力和窗口，绝不保存 endpoint、Header、API Key 或环境变量值。
- 复用 ZC-071 的受控配置写入服务，只允许修改白名单中的 `models.default_profile`；需要处理 compare-and-swap、配置来源、策略锁、原子写入和重启后的读取一致性。
- 扩展 ModelRouter、RuntimeProfile 与 JSON-RPC，使当前 Thread 后续 run 使用新 executor Profile；旧 Runtime 依照既有 lease/DRAINING 语义释放，不取消已完成历史，也不复用错误模型的 Runtime。
- `/model` Picker、已选提示、失败状态和焦点恢复应明确区分“切换当前 Thread”“设为后续新 Thread 默认”“历史 Thread 的既有绑定”。

## 非范围

- 不允许活动 run、待审批、待问答或取消收敛期间热切换模型。
- 不将当前全局选择强制应用到恢复的历史 Thread；恢复时继续使用该 Thread 最后一次有效绑定。
- 不修改 API Key、Provider endpoint、Header、Role mapping 或任意非白名单配置；不实现多 Agent Planner→Executor 调度、`/agent` 切换或模型自动选择。
- 不接入新的厂商 SDK、模型市场或远程配置同步。

## 验收清单

- 空闲当前 Thread 切换后下一次 run 实际使用选择的 Profile；新建 Thread 和重启后的新建 Thread 继承最后选择；再次切换能覆盖默认并形成新的绑定事件。
- 恢复旧 Thread 时仍使用其最后有效绑定；模型切换前后的 run、RuntimeProfile、Thread 摘要与审计历史相互一致，且所有 DTO 均脱敏。
- 无效/不可用 Profile、配置写入冲突、策略锁、磁盘失败、活动 run、审批、问答、取消、RPC 重试及窄终端/焦点恢复均失败闭合并保持原选择。
- Python 覆盖配置原子写入、绑定历史、路由和 RuntimePool 生命周期；TypeScript 覆盖 Picker、当前 Thread 即时生效、新 Thread 继承、恢复语义和错误状态；协议、类型、全量测试与文档检查通过。
