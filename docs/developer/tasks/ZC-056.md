## 范围

- 实现 `AgentRuntime`、`AgentRuntimeLease`、`RuntimePool` 及每个 Profile 的 single-flight 构建协调；Runtime 持有编译图、图级工具/资源适配器和关闭责任，Thread 不持有完整 Agent。
- 实现 lease、active run、queued run、last used、pin 和状态转换记录；DRAINING Runtime 拒绝新的 acquire，旧 lease 可按策略完成或取消。
- 提供幂等 `aclose()`，依次停止接收新 run、等待/取消后台任务、刷新持久状态、关闭 Tool/MCP/Sandbox/Runtime 独占客户端并释放图引用；单项关闭失败必须记录并继续后续清理。
- 抽象资源 bundle 和关闭适配器，使当前本地实现可无资源降级，未来 MCP、Sandbox、Agent Team 不依赖模块级全局变量。

## 非范围

- 不接入 JsonRpcServer 生产请求路径，不定义池容量、TTL 或内存压力策略。
- 不新增 MCP、Sandbox、Provider 或 Agent Team 功能。
- 不关闭进程级 ProviderClientPool 将来拥有的共享 HTTP client。

## 验收清单

- 100 个并发 acquire 同一 Profile 时，构建工厂只调用一次；构建失败后下一次 acquire 可以重试。
- lease 或 active run 存在时 evict 被拒绝或延迟；进入 DRAINING 后新请求不会取得旧 Runtime。
- `aclose()` 可重复调用，且在某个 mock 资源关闭失败后仍关闭其余资源并释放图引用。
- 单元测试覆盖 MISSING、BUILDING、READY、ACTIVE、IDLE、DRAINING、CLOSED 和失败恢复路径。
