## 范围

- 将 `/resume [query]` 由单纯恢复选择器演进为 Thread Manager：显示稳定摘要、当前状态、重命名、删除、导出和保留策略；搜索参数只预填筛选，不接收内部 ID。
- 删除、导出和保留策略必须显示影响范围并确认；导出复用 ThreadStore/Checkpointer 的当前 project 读取边界，内容与格式受用户可读性和脱敏要求约束。
- 为 ZC-074 的时间线/fork/rewind 暴露稳定的 thread 摘要与生命周期接口，但本任务不实现分支或回退。

## 非范围

不做云同步、跨 project 恢复或重新引入 session 概念。

## 验收清单

- 删除、导出或保留策略必须显式确认，并只作用于当前 project 的 thread。
- 新入口与现有 `/resume` 选择器共享稳定摘要，不要求用户输入或显示 `thread_id`。
- 重命名、删除、导出、保留策略、损坏记录和并发读取有端到端测试。
