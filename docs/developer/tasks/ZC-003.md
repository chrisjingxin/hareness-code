## 范围

交付 `~/.harness/threads.sqlite3`、`harness_threads` 索引、project SHA-256 隔离、`threads.read`/`threads.list`/`threads.open`、交互式 `--resume`、`/resume` 与悬浮恢复选择器。术语固定为 project、thread、message，身份字段固定为 `thread_id`。

## 非范围

不设计云端 thread 同步、删除、导出、跨 project 恢复、模型切换、LocalContext 或上下文压缩策略。

## 验收清单

- 进程重启后，使用同一 `thread_id` 的下一轮能读取此前消息。
- SQLite 连接、关闭、迁移、project 隔离和损坏数据库的失败路径有 Python 回归测试。
- `harness --resume` 和 `/resume` 都打开可搜索选择器；键盘、鼠标、Esc 和消息历史回放通过 TUI 回归测试。
- 文档说明本地数据位置、project 隔离、保留范围和手动清理方式。
