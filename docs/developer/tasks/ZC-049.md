## 范围

- 流式工具分片以真实 `tool_call_id` 为权威；新的调用不得因 LangChain 重用 `index=0` 而回写旧调用。
- TUI 将用户消息、文本、工具、审批和问答按协议 sequence 渲染为单一时间线，并为每条工具调用保留独立输入与输出。
- 审批和问答解决后保留处理结果；活动状态只在时间线尾部表达当前进度。

## 非范围

- 不改变 JSON-RPC schema、审批策略、工具权限或新增会话 replay。
- 不复制 OpenCode 或 MiMo Code 的实现、依赖或视觉资源。

## 验收清单

- 多轮、同索引、跨 run 的工具流在 Python 和 TUI 状态测试中保持独立。
- TUI 渲染测试覆盖工具结果、内联审批与恢复活动行的顺序。
- 真实 PTY 在紧凑和常规终端尺寸下不发生卡片覆盖或底部布局重叠。
