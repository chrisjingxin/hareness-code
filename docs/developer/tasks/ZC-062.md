## 范围

- 实现 Coordinator、已登记 worker、任务队列和有界并发调度；所有角色、模型绑定、工具集和 sandbox 均由 RuntimeProfile/AgentTopology 预先声明。
- 通过 TeamStore/Mailbox 调度 direct、result、handoff、cancel 等消息，处理 worker 超时、失败、重试、完成、取消与 Coordinator 故障恢复。
- 将用户取消、同 thread 互斥、审批和问题交互聚合到主 run：worker 不得绕过 PolicyEngine、ToolScheduler 或主 thread 的批准范围。
- 将可展示的 team 生命周期投影到持久 EventLog/replay，并在 TUI 显示来源、状态与有限摘要；内部消息和大 payload 不直接暴露。

## 非范围

- 不支持任意网络地址的外部 Agent、自主创建新 Team、无限 fan-out、跨用户协作或多租户队列。
- 不实现跨设备实时同步、自由市场插件或让 Agent 修改 Policy/模型配置。
- 不替代 ZC-027 的受控远端 Agent 注册；远端 worker 仅可在其完成后作为一个受限适配器接入。

## 验收清单

- mock Team 在有界并发下完成多 worker 任务，Coordinator 能汇总结果并保留每个 worker 的来源与终态。
- 取消、拒绝审批、单个 worker 失败、超时、Runtime DRAINING 与 sidecar 重启后恢复均无悬挂 task、重复工具执行或消息丢失。
- worker 不能访问主角色未获准的工具、工作区、MCP、Sandbox 或凭据；跨 thread/team 消息被拒绝。
- 协议/TUI 回归只展示受限摘要，replay 不会重新投递内部 mailbox 或重新请求已解决审批。
