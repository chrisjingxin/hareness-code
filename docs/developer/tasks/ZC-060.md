## 范围

- 定义 `AgentTopology`、角色图/模板 registry 和 topology version，使 Single Agent、Planner/Executor 等图结构可由 RuntimeProfile 稳定选择。
- 实现保持兼容的单 Agent adapter，并实现一个受限的串行 Planner → Executor 流程；计划 Agent 使用 Pro，执行 Agent 使用 Fast，但两者共享同一 ThreadStore、RunContext、取消和审批边界。
- 规定 Agent 间交接的结构化字段、长度限制、来源标识和失败回退；交接数据属于 run/checkpoint 状态而非 Runtime 实例。
- 复核工具权限：worker 只能通过既有 ToolScheduler/审批策略执行，不能因角色切换获得额外权限。

## 非范围

- 不实现并行 worker、Agent 间自由通信、持久 mailbox 或 TeamScheduler。
- 不支持用户定义任意执行 DAG、无限递归子 Agent 或自治提权。
- 不修改远端子 Agent 的端点协议。

## 验收清单

- topology 变更形成新 Runtime Profile，旧 Runtime 在活跃 run 结束后按既有 DRAINING 语义释放。
- 单 Agent 默认路径的协议、工具调用、线程恢复和测试行为不变。
- Planner 输出只能以校验过的结构化交接进入 Executor；取消、拒绝审批、计划失败和执行失败均有明确终态且不串 thread。
- 两个角色的模型选择、调用顺序、权限复用和 checkpoint 恢复均有 mock 集成测试。
