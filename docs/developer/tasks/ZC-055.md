## 范围

- 定义只存活于一次调用的 `RunContext`，显式传递 `thread_id`、run ID、Profile Key、PromptEpoch、审批模式、取消令牌和当前 Agent 角色等运行信息。
- 将 PromptEpoch 的读取和渲染改为按 run 执行；共享图只保存模板和渲染逻辑，不能在闭包、middleware 实例或虚拟文件 backend 中固化某个 thread 的 epoch。
- 将 ContextWindowMiddleware 的可恢复状态收敛到 ThreadStore/Checkpointer 或无状态服务，提供不依赖 `_context_middlewares[thread_id]` 的强制压缩入口。
- 复核虚拟文件、审批、上下文归档和 LangGraph configurable state 的 thread 路由，消除共享图下的可变 thread 私有成员。

## 非范围

- 不实现 RuntimePool、资源淘汰或模型客户端复用。
- 不改变现有的 PromptEpoch 内容策略、压缩阈值、审批策略或 JSON-RPC 方法语义。
- 不实现 Pro/Fast 路由、Agent Team 或跨 thread 的状态共享。

## 验收清单

- 一个图实例服务 Thread A/B 的并发与交错执行时，消息、PromptEpoch、context artifact、审批决定和取消状态完全隔离。
- 复用图后 `/compact` 仍只改写目标 thread，且服务重启或图重建后可恢复同一结果。
- 生产代码和测试中不再以 `thread_id -> ContextWindowMiddleware` 保存运行状态。
- Python 回归覆盖共享图、强制压缩、取消和 thread 恢复的组合场景。
