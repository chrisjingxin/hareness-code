## 范围

- 在用户级 `~/.harness/config.toml` 的默认模型 Profile 中接受可选 `api_key`。
- 非空 `api_key_env` 环境变量始终优先，缺失或空白时才使用 TOML 密钥。
- Unix 上含字面量密钥的文件禁止 group/other 权限；Windows 不校验 POSIX 权限位。
- `config show` 仅显示 `environment` / `toml` / `missing` 来源和已配置状态。

## 非范围

- 不允许显式 `--config` 或项目 TOML 保存明文密钥。
- 不扩展认证 Header、租户 Token、MCP 或远端沙箱秘密。
- 不实现 Keychain、`.env`、密钥写入命令或配置热重载。
- 本任务不修改 `0.1.0` 版本。

## 验收清单

- 环境优先、TOML 降级、全缺失和空白密钥均有 Python 回归。
- 配置来源与 Unix 文件权限边界有回归。
- 脱敏摘要、对象 `repr`、网关适配器和本地后端不泄漏密钥。
- 用户模板和安全/配置文档明确 TOML 仅是低安全性降级方案。
