## 范围

- 定义 `CommandResult` 与 Dispatcher，将本地 Action、RPC 调用、notice、打开选择器、提交 Prompt 和退出统一成受类型约束的结果。
- 迁移 `/help`、`/quit`、`/clear`、`/force-clear`、`/compact`、`/status`、`/version`、`/resume` 和 `/skills`，由 handler 执行业务校验，TUI 根组件只适配结果。
- 引入 `/new` 作为创建 thread 的 canonical 命令，`/clear` 为别名；活动 run 下通过确认流程处理取消，`/force-clear` 在兼容期仅提示迁移，不能再形成第二套清理逻辑。
- 保持 `/resume` 不要求或显示 `thread_id`，保持既有 Skill 选择和 `context.compact` 协议行为。

## 非范围

- 不实现新的 Model、MCP、Settings、Diff 或 Workspace 命令。
- 不在本任务改变 `context.compact` 参数或新增协议 capability。

## 验收清单

- 现有命令的 handler、Dispatcher 和 Result Adapter 有单元与 OpenTUI 交互回归；取消失败和 RPC 失败显示可恢复通知。
- `/new`、`/clear`、废弃 `/force-clear`、`/resume`、`/continue` 的菜单、帮助和执行路径一致且无幽灵分支。
- `app.tsx` 不再保留按命令名 dispatch 的业务 switch，类型检查与现有 TUI 测试通过。
