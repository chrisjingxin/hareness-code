## 范围

- 在 TOML v1 定义 `architect`、`executor`、`reviewer`、`tester`、`summarizer` 等角色到命名模型 Profile 的绑定，并提供兼容的单模型默认映射。
- 实现 `ModelRouter` 和受限 `ProviderClientPool`：角色绑定属于 Runtime Profile，底层 HTTP/Provider client 可在 sidecar 生命周期复用，Profile 不得持有其他 Profile 的可变请求状态。
- 校验模型 Profile 是否支持当前所需工具、流式和上下文能力；将明确的能力/错误分类接口留给后续 Provider 扩展。
- 记录脱敏的实际角色/Profile 摘要，保证恢复和 Profile 变更不会静默使用与记录不一致的模型配置。

## 非范围

- 不接入 Anthropic、Gemini、Ollama 等新 Provider 协议，不改变 API Key 存储规则。
- 不实现运行中任意切换模型、自动成本优化或由模型自行选择 Provider。
- 不实现 Agent Team 编排；本任务只提供角色解析与路由能力。

## 验收清单

- 现有单模型配置无修改继续运行；合法 Pro/Fast 绑定为每个角色解析正确 Profile。
- 未知角色、缺失 Profile、能力不足和秘密 header 传播均有失败闭合测试。
- 角色绑定改变必然产生新的 Runtime Profile，不同 thread 不串用模型或请求状态。
- mock Provider 验证同一底层 client 被安全复用，且诊断只显示脱敏 Profile 信息。
