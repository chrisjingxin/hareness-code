# 模型配置

v0.1 只支持 OpenAI 兼容协议。TOML 使用 v1 结构；当前只读取用户配置 `~/.harness/config.toml` 和用户明确传入的 `--config <PATH>`。覆盖优先级为 `--sandbox` 等 CLI 参数 > `HARNESS_*` 环境变量 > `--config` > 用户配置 > 默认值。

## 配置文件

从仓库中的 [用户级示例配置](examples/config.toml) 开始：

```toml
[config]
version = 1

[models]
default_profile = "enterprise"

[models.profiles.enterprise]
provider = "openai-compatible"
model = "your-enterprise-coding-model"
base_url = "https://gateway.example.internal/v1"
api_key_env = "HARNESS_API_KEY"
```

`api_key_env` 填写环境变量名，而不是密钥值。API Key、令牌和敏感租户信息不得写入 TOML 或提交到仓库。

```bash
export HARNESS_API_KEY='你的企业网关密钥'
bun run dev -- config path
bun run dev -- config show
```

`config show` 仅展示脱敏后的有效配置。可用环境变量包括 `HARNESS_MODEL`、`HARNESS_BASE_URL`、`HARNESS_API_KEY_ENV`、`HARNESS_TIMEOUT_SECONDS`、`HARNESS_MAX_RETRIES`、`HARNESS_APPROVAL_MODE` 和 `HARNESS_SANDBOX`。`HARNESS_AGENT_PYTHON` 仅用于开发时定位 sidecar，不属于 TOML 配置字段。

企业网关若需要非秘密固定头，可用 `[models.profiles.<name>.headers]`；若头值来自环境变量，使用 `[models.profiles.<name>.headers_env]`。认证 Header、Cookie 和 API Key 只能使用 `headers_env`，不能写入 TOML。当前只允许一个默认 Profile，多 Profile 选择尚未实现。

工作区 `.harness/config.toml` 和 `.harness/config.local.toml` 当前不会被自动读取。检测到这两个文件时，Harness Code 会在发起模型请求前失败；请移至 `~/.harness/config.toml`，或由你主动传入 `--config <PATH>`。

工具执行默认在本机。企业远端 sandbox 的配置、启用方式和限制见 [安全与沙箱](安全与沙箱.md)。
