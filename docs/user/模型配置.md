# 模型配置

v0.1 只支持 OpenAI 兼容协议。配置优先级为：用户配置 `~/.harness/config.toml`、工作区 `.harness/config.toml`、`ZA38_*` 环境变量、`--config <PATH>` 显式配置；后者覆盖前者。

## 配置文件

从仓库中的 [示例配置](../../.harness/config.toml.example) 开始：

```toml
[model]
provider = "openai-compatible"
name = "your-enterprise-coding-model"
base_url = "https://gateway.example.internal/v1"
api_key_env = "ZA38_API_KEY"
timeout_seconds = 120
max_retries = 2
```

`api_key_env` 填写环境变量名，而不是密钥值。API Key、令牌和敏感租户信息不得写入 TOML 或提交到仓库。

```bash
export ZA38_API_KEY='你的企业网关密钥'
za38 config path
za38 config show
```

`config show` 仅展示脱敏后的有效配置。可用环境变量包括 `ZA38_MODEL`、`ZA38_BASE_URL`、`ZA38_API_KEY_ENV`、`ZA38_TIMEOUT_SECONDS` 和 `ZA38_MAX_RETRIES`。

企业网关若需要非秘密固定头，可用 `[model.headers]`；若头值来自环境变量，使用 `[model.headers_env]`。完整例子见示例配置。

工具执行默认在本机。企业远端 sandbox 的配置、启用方式和限制见 [安全与沙箱](安全与沙箱.md)。
