# 故障排查

## 找不到模型配置或 API Key

确认 `.harness/config.toml`、`~/.harness/config.toml` 或 `--config` 指向的文件包含 `[model]`，并为 `api_key_env` 指定的环境变量设置密钥。使用以下命令确认生效配置；输出不会包含密钥：

```bash
za38 config path
za38 config show
```

## Python sidecar 无法启动

开发模式优先使用 `packages/agent/.venv/bin/python`。确认虚拟环境存在、Python 版本为 3.11+，并在仓库根目录运行命令。需要临时指定解释器时设置 `HARNESS_AGENT_PYTHON`。

使用 `--cwd <路径>` 时，该目录必须已经存在；CLI 不会自动创建工作区。显式配置路径建议使用绝对路径，避免它随 Agent 工作目录变化：

```bash
mkdir -p /private/tmp/za38-smoke
bun run dev -- --cwd /private/tmp/za38-smoke \
  --config "$PWD/.harness/config.toml"
```

## 终端界面显示简化

在窄终端（小于 88×28）或 `TERM=dumb` 下，星空与品牌动画会主动关闭以保证可读性。请扩大终端，或使用支持 ANSI 色彩的终端模拟器。

若界面出现 `^[[`、`rgb:`、`Warp(...)` 或鼠标坐标等控制序列，说明 TUI 被 workspace runner、管道或输出捕获器启动。请在仓库根目录使用 `bun run dev -- ...`；不要使用 `bun run --filter '@za38/cli' dev` 启动交互界面。

## 网关请求失败

检查 `base_url` 是否为企业 OpenAI 兼容接口根路径、模型名是否可用、企业代理和必要请求头是否已配置。切勿把真实密钥贴进 Issue、终端截图或仓库文件。

## 协议版本或序号错误

CLI 与 Python sidecar 必须来自同一发行版本。出现 `PROTOCOL_MISMATCH` 时，检查是否混用了旧的 Python 环境或 CLI 构建产物；源码开发时重新运行 `bun run build` 并确认 `HARNESS_AGENT_PYTHON` 指向本仓库虚拟环境。

会话中出现“协议序号缺口”表示同一运行的事件不连续。界面会继续处理后续事件，但应保留 stderr 和终端截图用于诊断。不要把包含密钥、工具完整参数或企业路径的原始帧直接提交到公开渠道。

如果问题涉及尚未实现的能力，请先查看 [任务看板](../developer/任务看板.md)；若不是已知待办，请按 [开发工作流](../developer/开发工作流.md) 提供最小复现和日志摘要。
