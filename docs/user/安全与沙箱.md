# 安全与沙箱

Harness Code 默认在本机执行，与 Qwen Code 未开启 sandbox 时一致。`--cwd` 决定本机工作区：内置 `ls`、`read_file`、`write_file`、`edit_file`、`glob` 和 `grep` 只能访问该目录内的文件。

直接文件工具必须使用工作区内的绝对路径。`glob` 和 `grep` 可以省略搜索目录，此时从工作区搜索；显式目录同样必须位于工作区内，文件匹配模式不能使用绝对路径或 `..`。Harness Code 会在工具执行前解析符号链接，因此工作区内指向外部的链接也不能用来读取或写入外部文件。此限制不能通过审批绕过。

这不是操作系统隔离或完整沙箱：`execute`、MCP 工具和未来自定义工具仍可能具有各自的文件访问能力，并按其审批或企业策略执行。终端状态显示“本机执行 · 未隔离”时，请仅在可信开发环境中使用。

不要将 API Key、云凭据或其他秘密放在工作区中。模型 API Key 推荐通过环境变量提供；如必须使用明文 TOML 降级值，只能写入用户级 `~/.harness/config.toml`。macOS/Linux 上 Harness 会拒绝读取存在 group/other 权限的明文密钥文件；建议保持目录 `0700`、文件 `0600`。

## 审批模式

审批模式只在启动时解析，不能在 thread 中切换。请在用户配置 `~/.harness/config.toml`、显式传入的 `--config` 文件，或环境变量 `HARNESS_APPROVAL_MODE` 中设置；优先级依次为用户配置 < 显式配置 < 环境变量。当前项目目录的 `.harness/config.toml` 和 `.harness/config.local.toml` 完全不支持，检测到即要求迁移。

```toml
[approval]
mode = "default"
```

| 模式 | 文件编辑 | Shell、删除、子 Agent | 上下文压缩 |
| --- | --- | --- | --- |
| `plan` | 拒绝 | 拒绝 | 自动执行 |
| `default` | 每次确认 | 每次确认 | 自动执行 |
| `auto-edit` | 自动执行 | 每次确认 | 自动执行 |
| `yolo` | 自动执行 | 自动执行 | 自动执行 |

`plan` 只允许 `ls`、`read_file`、`glob`、`grep`、`ask_user` 和 `write_todos`。写入、`execute`、`task` 以及未识别的未来 MCP 工具都会直接返回拒绝结果，不会弹出审批框。

Harness 还提供只读的 `/.harness/` 逻辑命名空间。它不是本机文件路径：只有 `read_file` 可以按页读取 Skill 正文、Skill 资源或本 thread 已归档的历史；`write_file`、`edit_file`、`ls`、`glob`、`grep` 和 shell 命令都会被拒绝。该路径不会泄露 Skill 的宿主目录或 SQLite 文件位置。

`yolo` 只关闭 Harness Code 自身的人工确认。它不会绕过工作区边界、Shell 白名单、远端 sandbox provider 或企业安全策略。旧值 `ask` 会兼容为 `default`；空值或未知值同样安全降级为 `default`，TUI 会显示对应提示。

## 启用企业远端沙箱

企业提供远端 sandbox 插件后，在用户配置 `~/.harness/config.toml` 或用户明确传入的 `--config` 文件中设置：

```toml
[approval]
mode = "default"

[execution]
backend = "remote"

[execution.remote]
provider = "corp"
factory = "harness_corp_sandbox:create_backend"
working_directory = "/workspace"

[execution.remote.params]
project = "your-enterprise-project"
```

也可以单次执行：

```bash
za38 --sandbox
HARNESS_SANDBOX=true harness
```

远端 provider 负责认证、工作区同步、网络策略和生命周期，并必须实现 deepagents 的 `SandboxBackendProtocol`。沙箱不可用时命令会失败，不会自动回退到本机执行。

`sandbox.params` 只传递非秘密 provider 参数；认证信息必须由企业身份机制或环境变量处理，不得写入 TOML。

当前不提供 Docker、Podman 或本机容器 sandbox。企业网络白名单由远端 provider 管理。
