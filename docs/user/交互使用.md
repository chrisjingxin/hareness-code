# 交互使用

启动 `za38`（源码开发阶段可用 `bun run dev --`）后，直接输入编码任务。Harness Code 会将流式文本、工具调用、审批请求和问题显示在同一 thread 时间线中。

企业 OpenTUI Core/React `0.4.3` 使用随 CLI 分发的离线 Tree-sitter parser 与语法查询渲染 Agent Markdown。标题、强调、列表、引用和 fenced code 会按终端样式显示；代码高亮不在运行时下载任何资源。

## 执行时间线

每次工具调用都是独立的历史事件。工具卡会显示本次调用的参数预览和实际结果；即使 Agent 连续调用同一个工具，先前结果也不会被后续调用更新。

需要审批或回答问题时，交互卡会停留在触发它的位置；完成后会显示已允许、已拒绝或已回答。思考、恢复执行和取消等当前状态只显示在时间线末尾，因此不会插入或覆盖已经完成的工具结果。

## 快捷键

| 快捷键 | 行为 |
| --- | --- |
| `Enter` | 发送输入 |
| `Shift+Enter` | 输入换行 |
| `/` 或 `Ctrl+P` | 打开本地 Slash Command 菜单 |
| `Esc` | 关闭命令菜单或取消当前运行 |
| `Ctrl+C` | 优先清空输入，其次取消运行，最后退出 |
| `Ctrl+O` | 展开或折叠工具详情 |
| 空输入时 `↑` / `↓` | 回填历史提示词；没有历史时浏览 thread |
| 空输入时 `PageUp` / `PageDown` | 按半页浏览 thread |

## 已实现的 Slash Command

- `/help`：显示命令说明。
- `/clear`：清空当前 thread 并回到首页。
- `/force-clear`：取消执行后清空当前 thread。
- `/compact`：在当前 thread 空闲时手动压缩上下文。它保留最近两轮、将更早历史写入本机归档，并在没有足够历史或预计节省不足 20% 时不改写会话。
- `/status`：显示工作区、模型、执行后端与审批模式。
- `/version`：显示 CLI 版本。
- `/resume`：打开当前 project 的悬浮 thread 选择器。输入关键词过滤消息摘要，使用方向键或鼠标选中，按 Enter 恢复完整消息历史；Esc 关闭且不改变当前输入。正在执行或等待交互时不能切换 thread。
- `/skills`：在当前 thread 上打开可搜索的悬浮 Skill 选择器。输入关键词过滤目录，使用方向键或鼠标选中，按 Enter 后 Skill 会附到下一条消息；Esc 关闭且不改变当前输入。
- 在 composer 输入 `/`：命令补全菜单同时显示 `skill:<canonical-id>`。筛选并选中任意 Skill 后，效果与 `/skills` 相同。
- `/quit` 或 `/q`：退出。

项目 `<workspace>/.harness/skills/<name>/SKILL.md` 默认生效，用户 Skill 位于 `~/.harness/skills/<name>/SKILL.md`（也兼容 `~/.harness/skills/local/<name>/SKILL.md`）。同名 Skill 不会互相覆盖，冲突时使用完整 canonical ID，例如 `project/review`。需要暂时停用时运行 `harness skills disable project/review`，重新启动 Harness 后生效。

命令行管理入口：

```bash
harness skills list
harness skills inspect project/review
harness skills enable project/review
harness skills disable project/review
harness skills trust project/review  # enable 的兼容别名
harness skills market --market enterprise
```

市场安装、更新和删除只调用企业 Provider 接口；未安装 Provider 时会返回明确诊断，不会访问公网。

`/threads`、`/mcp`、`/tools`、`/agents` 和 `/remember` 尚未接入后端；请使用 `/resume` 恢复 thread。

## 恢复 Thread

运行 `harness --resume` 会直接打开与 `/resume` 相同的选择器。选择器只显示消息摘要、更新时间和消息数；内部 `thread_id` 不会显示，也不需要手动输入。恢复范围始终是当前 project，不能浏览或恢复其他 project 的记录。

Harness 将可恢复 thread 和 LangGraph checkpoint 存在用户目录的 `~/.harness/threads.sqlite3`。目录权限为 `0700`，新建数据库权限为 `0600`；数据库只保存 project 规范路径的 SHA-256 指纹，不保存原始 project 路径。Harness 停止后，可以手动删除 `threads.sqlite3`、`threads.sqlite3-wal` 与 `threads.sqlite3-shm` 清理全部本地 thread。

模型配置、无头调用和配置查询见 [模型配置](模型配置.md) 与 [快速开始](快速开始.md)。
