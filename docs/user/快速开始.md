# 快速开始

Harness Code 当前以源码开发运行。请使用 Bun `1.2.19`、Python 3.11+ 和 `uv`；企业 CI 或本地环境通过 `UV_INDEX_URL` / 凭据环境变量提供 Python 私有索引，索引地址和凭据不会写入仓库。

## 配置并启动

在仓库根目录将模板复制到用户级配置目录，并提供模板声明的 API Key 环境变量：

```bash
cd packages/agent && uv sync --extra test && cd ../..
mkdir -p ~/.harness
cp docs/user/examples/config.toml ~/.harness/config.toml
export HARNESS_API_KEY='你的企业网关密钥'
bun run dev
```

交互启动后直接输入任务即可。首次发送后进入会话视图；输入 `/` 可打开已实现的命令菜单。

## 无头模式

适合脚本或本地验证：

```bash
bun run dev -- --non-interactive --message "总结这个仓库"
bun run dev -- --non-interactive --message "列出风险" --json
```

使用 `--cwd <路径>` 指定工作区，使用 `--resume <thread_id>` 尝试恢复已知线程。持久化会话管理尚未实现，`--resume` 仅能在当前后端状态仍可用时使用。

安装器、自动更新和跨平台预编译包仍是待交付能力，详见 [开发任务看板](../developer/任务看板.md)。

## OpenTUI 依赖

源码运行依赖企业源提供的 `@opentui/core` / `@opentui/react` `0.4.3`。企业引入 Core 时还必须镜像其当前平台对应的 `@opentui/core-<platform>-<arch>` optional package；安装后由 Bun 按运行平台选择，不需要本地 Zig 或源码快照。
